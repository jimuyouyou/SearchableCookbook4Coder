General solution for avoiding Javascript memory leak

For some devices, like mobile phone, pad and TV, there are many reasons that can crash your web pages:
  • Multile loading of static files 
    	common static files includes: javascript, css and pictures
	Almost all browsers will automatically cache these files and they will not download static files for the second time. However, some apps would like to add a mark to url link for forcing the browser to load the latest static files. This is not necessary. 

	examples: good.jpg?v=34, main.css?date=04, dashboard.js?20150506
	Solution: remove all the characters after '?'
